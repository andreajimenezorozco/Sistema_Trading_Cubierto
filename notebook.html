Diferenciación Fraccional
¶

  • Una prctica comn en ML es pre-procesar los datos para que la serie con la que se va a trabajar sea estacionaria (media y varianza constante)

  • La forma ms popular de convertir una serie no estacionaria en estacionaria es mediante las diferencias, es decir los rendimientos (naturales o logartmicos)

  • Al hacer esto, se pierde casi por completo la memoria de la serie

  • Las series financieras tienen memoria, lo cual es importante considerar si lo que queremos es hacer predicciones sobre ella

  • Entonces, si ganamos estacionariedad perdemos memoria, pero si conservamos memoria no tenemos estacionariedad

Habr una manera de diferenciar, tal que obtengamos una serie estacionaria y conservemos la memoria de la serie original?

Uniendo los conceptos de expansin mediante Serie de Taylor, y el operador rezago $B$ donde $B^{k}X_{t} = X_{t-k}$ obtenemos lo siguiente:

$(1-B)^{d} = 1 - dB + \frac{d(d-1)}{2!}B^{2} - \frac{d(d-1)(d-2)}{3!}B^{3} + ...$

Donde $(1-B)^{d}$ es la diferenciacin de una serie y $d$ representa el factor de diferenciacin que estamos aplicando


Por lo tanto, los coeficientes de cada $B^{n}$ se pueden expresar como

$w_{k} = -w_{k-1}\frac{d-k+1}{k}$

Lo cual nos permite utilizar un numero entre $(0,1)$ para el grado de diferenciacin $d$

Change Point Detection

Anlisis estadistico, su funcin es detectar los puntos en donde el proceso estcastico o la serie de tiempo cambia.

  • Mtodos: Window, binary, PELT.

  • Resultado de los mtodos: Anomalas dentro de la serie de tiempo, fechas en donde ocurri el cambio y lista de variables dummy, donde 1 es un change point y el 0 sin anomala.

  • Entradas: Dataframe que contenga una columna close.

  • Librera: ruptures

  • Window: se utiliza para realizar una segmentacin rpida de la seal. El algoritmo utiliza dos ventanas que se deslizan a lo largo del flujo de datos. Las propiedades estadsticas de las seales dentro de cada ventana se comparan con una medida de discrepancia.
  • Binary: La segmentacin binaria es el mtodo estndar a lo relacionado a change point detection. Bsicamente, aplica iterativamente el change point a diferentes subconjuntos de la secuencia y1.,yn con el objetivo de detectar mltiples puntos de cambio.
  • PELT: Es un mtodo exacto y genera resultados rpidos y consistentes. Minimiza la funcin de costo sobre posibles puntos de cambio. Este mtodo tiene costo computacional lineal a cada uno de los puntos dentro de la secuencia de precios. Para tener una mayor potencia del mtodo se necesitarn un nmero mayor de datos. Si un cambio es muy grande, existen mayores posibilidades de que el mtodo encuentre ese punto exacto.

    df_pe['quartiles'] = Add_features.quartiles(df_pe,10)

def createNN(neuron_pctg,lr,layer_pctg):
    model = Sequential()
    nneurons = float(neuron_pctg) * 100  
    nlayers = float(layer_pctg)*75
    layer_counter = 0
    model.add(Dense(115,activation="sigmoid",input_shape=(114,))) 
    while nneurons<=10 and layer_counter < nlayers:
        model.add(Dense(nneurons, activation="sigmoid"))
        model.add(Dropout(0.5))
        layer_counter += 1
    model.add(Dense(1,activation="sigmoid"))
    opt = keras.optimizers.Adam(learning_rate=lr)
    model.compile(loss='binary_crossentropy',optimizer=opt,metrics=["accuracy"])
    return model
